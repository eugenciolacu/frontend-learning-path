Semantic vs Non-Semantic Structure

Compare the meaning communicated by generic containers and semantic elements.

Non-Semantic Version

<div class="page-header">
	<div class="menu">...</div>
</div>
<div class="content">...</div>
<div class="sidebar">...</div>

This version can be styled, but the element names do not explain the page regions.

Semantic Version

<header>
	<nav>...</nav>
</header>
<main>...</main>
<aside>...</aside>

This version clearly explains the role of each region before any CSS is added.

Key Takeaway

Use div and span when you truly need generic containers, but prefer semantic elements when HTML already provides the correct meaning.
