Accessible Tooltip Web Component

The highlighted terms below are rendered with a custom HTML element named learning-tooltip. Each instance uses a shared HTML template and renders its own encapsulated structure inside Shadow DOM.

Glossary Terms

Move with the keyboard using Tab or hover with a pointer to reveal each explanation.

  • Semantic HTML helps browsers and assistive technologies understand page meaning.
  • Shadow DOM keeps the tooltip internals isolated from the surrounding document.
  • Custom Element turns repeated UI into a reusable browser primitive.
  • Focus Management makes the component usable without a mouse.

Tooltip Examples in Context

This mini-project uses native HTML first. The page structure is built with semantic landmarks, while the reusable interaction is isolated in a Web Component.

Document Structure

A clear heading hierarchy and page landmarks make the content easier to scan. Use landmarks to communicate page regions.

Component API

Each tooltip exposes content through attributes such as tooltip-title, tooltip-text, and position. That makes the component easy to reuse in plain HTML.

Keyboard Support

When the trigger receives focus, the tooltip opens. Press Escape to close it and keep navigating.

Accessibility Checklist

  1. Each tooltip trigger is a real button, not a generic div.
  2. The component provides visible focus states for keyboard users.
  3. The tooltip panel uses role="tooltip" and connects to the trigger with aria-describedby.
  4. The page includes a skip link and semantic landmarks.

Study Notes

This project stays within Module 1 knowledge: document structure from HTML foundations, semantic page architecture, accessibility guidance, and Web Components.

No framework, build tool, or external library is required. Open index.html in a browser to inspect the result.
