Comparing CSS Modules, CSS-in-JS, and utility-first styling

This browser-friendly page mirrors the same button component rendered through three architectural approaches. The source React snippets stay in this folder, while this HTML file provides a direct visual reference.

CSS Modules

Build-time local scoping

Styles are written in a regular CSS file and imported into the component. The bundler rewrites class names to keep them local.

import styles from "./Button.module.css";

export function Button({ tone = "primary" }) {
  const toneClass = tone === "secondary"
    ? styles.secondary
    : styles.primary;

  return <button className={`${styles.button} ${toneClass}`}>...</button>;
}

CSS-in-JS

Dynamic styling with props

Styles are colocated with component logic. This is useful when component state or props control visual output directly.

const StyledButton = styled.button`
  background: ${({ $tone }) =>
    $tone === "secondary" ? "#e2e8f0" : "#0f766e"};
  color: ${({ $tone }) =>
    $tone === "secondary" ? "#0f172a" : "#ffffff"};
`;

Utility-first

Token-driven classes in markup

Utility classes make visual decisions visible in the component markup. This reduces custom CSS but increases class density.

<button
  className="inline-flex items-center justify-center
             rounded-xl px-4 py-3 font-semibold
             bg-teal-700 text-white"
>
  Enroll now
</button>
